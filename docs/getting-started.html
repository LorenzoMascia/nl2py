Getting Started

This guide will help you install NL2Py and start converting natural language to Python code in minutes.

Installation

Requirements

  • Python 3.10 or higher
  • pip package manager

Install from PyPI

pip install nl2py

Install from Source

# Clone the repository
git clone https://github.com/lorenzomascia/nl2py.git
cd nl2py

# Create a virtual environment (recommended)
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install in development mode
pip install -e .

Install Optional Dependencies

# For the web GUI
pip install gradio>=4.0.0

# For specific modules (see requirements.txt for full list)
pip install boto3          # AWS
pip install google-cloud-storage  # GCP
pip install docker         # Docker
pip install psycopg2-binary  # PostgreSQL
Tip: Install only the dependencies for the modules you need to keep your environment lean.

Quick Start

After installation, you can start using NL2Py immediately:

# Launch the web GUI
nl2py-gui

# Or use Python directly
python -c "
from nl2py import create_interpreter
interp = create_interpreter()
result = interp.interpret('create s3 bucket my-data')
print(result.generated_code)
"

Web GUI

NL2Py includes a beautiful Gradio-based web interface.

Launch the GUI

# Basic launch (opens at http://localhost:7860)
nl2py-gui

# With custom port
nl2py-gui --port 8080

# Create a public shareable link
nl2py-gui --share

GUI Features

Tab Description
Single Command Translate one command at a time with match details
Full Translation Convert multiple lines into a complete Python script
Line-by-Line See detailed analysis for each line
Explore Matches Find top matching methods for a command
Methods Reference Browse all available methods with search

Adjusting Similarity Threshold

The threshold slider controls how strict the matching is:

  • 0.1 (default) - Good balance, catches most commands
  • 0.05 - More lenient, may have false positives
  • 0.3+ - Stricter, only high-confidence matches

CLI Usage

Interactive Mode

python -m nl2py.nlp_interpreter --interactive

# Example session:
> create compute instance my-server
1. GCPModule.compute_instance_create
   Score: 0.72
   Code: compute_instance_create(name='my-server')

> exit

Process a File

# Input file: commands.txt
# create s3 bucket data-lake
# list docker containers
# send slack message to devops

python -m nl2py.nlp_interpreter commands.txt output.py

# output.py will contain:
# from nl2py import modules
# s3_create_bucket(name='data-lake')
# container_list()
# send_message(channel='devops')

Python API

Basic Usage

from nl2py import create_interpreter

# Create and initialize
interpreter = create_interpreter()

# Interpret a single command
result = interpreter.interpret("create s3 bucket my-data")

# Access result properties
print(result.generated_code)    # s3_create_bucket(name='my-data')
print(result.module_name)       # S3Module
print(result.method_name)       # s3_create_bucket
print(result.similarity_score)  # 0.75
print(result.extracted_params)  # {'name': 'my-data'}

Get Multiple Matches

# Get top 5 matches for exploration
results = interpreter.match(
    "upload file to storage",
    threshold=0.05,
    top_k=5
)

for r in results:
    print(f"{r.module_name}.{r.method_name}: {r.similarity_score:.2f}")

Process Multiple Lines

from nl2py.nlp_interpreter import FileInterpreter

fi = FileInterpreter()

# Process a file
code = fi.process_file(
    "commands.txt",
    output_path="output.py",
    threshold=0.1,
    include_comments=True
)

# Or process a list of strings
lines = [
    "create bucket my-bucket",
    "upload file data.csv to my-bucket",
    "list bucket contents"
]
results = fi.process_lines(lines)

Configuration File

Create nl2py.conf to configure module credentials:

# nl2py.conf

[postgres]
host = localhost
port = 5432
database = myapp
user = postgres
password = secret

[aws]
region = us-east-1
access_key_id = AKIA...
secret_access_key = ...

[slack]
webhook_url = https://hooks.slack.com/services/...
bot_token = xoxb-...

[redis]
host = localhost
port = 6379
password =
Security: Never commit nl2py.conf to version control. It's gitignored by default.

Environment Variables

You can also configure modules using environment variables:

# AWS
export AWS_ACCESS_KEY_ID=AKIA...
export AWS_SECRET_ACCESS_KEY=...
export AWS_DEFAULT_REGION=us-east-1

# GCP
export GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json
export GCP_PROJECT_ID=my-project

# Database
export POSTGRES_HOST=localhost
export POSTGRES_USER=postgres
export POSTGRES_PASSWORD=secret

Writing Effective Commands

Tips for Better Matching

  1. Be specific - Include key words that describe the action 
    # Good
create compute instance web-server in zone us-central1-a

# Less specific (still works)
make a server called web-server
  2. Use common verbs - create, delete, list, get, send, upload, download 
    create bucket my-data
list all containers
send message to slack
  3. Include parameter names - Makes extraction more accurate 
    # With parameter hints
create instance named my-server in zone us-west-1

# Without (still works but less precise)
create my-server us-west-1

Example Commands by Category

# Cloud Infrastructure
create compute instance {{name}} in zone {{zone}}
create s3 bucket {{bucket-name}} in region {{region}}
list all virtual machines

# Databases
connect to postgres database {{dbname}}
execute query "SELECT * FROM users"
set redis key {{key}} to {{value}}

# Messaging
send message to slack channel {{channel}} with text {{message}}
publish to kafka topic {{topic}}
send email to {{recipient}} with subject {{subject}}

# Containers
list docker containers
start container {{name}}
deploy kubernetes deployment {{name}} with {{replicas}} replicas

Troubleshooting

No matches found

  • Lower the similarity threshold (try 0.05)
  • Use the "Explore Matches" tab to see what's close
  • Check the "Methods Reference" for available commands
  • Rephrase using more common verbs

Wrong method matched

  • Be more specific in your command
  • Include the service name (e.g., "s3 bucket" vs just "bucket")
  • Increase the similarity threshold

Parameters not extracted

  • Use quotes for multi-word values
  • Include parameter keywords like "named", "called", "in zone", "to"
  • Check the matched example's pattern in "Explore Matches"

Module not available

# Check if module dependencies are installed
pip show boto3  # For AWS
pip show docker  # For Docker

# Install missing dependencies
pip install boto3
